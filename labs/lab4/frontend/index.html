Registry status

Contract state

Issue result

Write transaction

A transaction hash, block number, nonce change, and count change prove that the issuer changed state.

Verification result

Read-only check

No transaction hash appears because verification reads state. It answers: not found, valid, or revoked.

Revoke result

Status change

Revocation keeps the record but flips validity. This protects audit history better than deletion.

Event history

Audit trail

Events help explain order: which certificate was issued or revoked, in which block, and by which transaction.
